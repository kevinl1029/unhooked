## 2026-02-15 17:45 EST - US-004
- Created prompt injection sanitization utility
- Files changed:
  - server/utils/personalization/sanitize-user-text.ts (created)
- **Learnings for future iterations:**
  - The sanitization utility is simple regex-based string processing
  - Applied at context builder level (NFR-4.3) to sanitize user-provided text before LLM prompt injection
  - Strategy: Strip LLM control patterns (role markers, markdown headers, triple backticks, XML tags), then wrap in delimiters in prompt template
  - Key regex patterns: `/^(SYSTEM|ASSISTANT|Human|AI|User):\s*/gmi` for role markers, `/^#{1,6}\s+/gm` for markdown headers, `/```/g` for backticks, `/<\/?[a-z][a-z0-9]*[^>]*>/gi` for XML/HTML tags
  - Preserves normal names like 'Kevin', 'Skip', 'Ignore' without false positives
  - File location follows pattern: `server/utils/personalization/` for personalization utilities
