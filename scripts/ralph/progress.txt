**Learnings for future iterations:**
- Migration pattern: Use `ADD COLUMN IF NOT EXISTS` for safe migrations that can run multiple times
- CHECK constraints: Use DO block to add constraints conditionally (prevents errors on re-run)
- Type patterns: Database types follow Row/Insert/Update structure with nullable fields having `| null` in Row and `?: type | null` in Insert/Update
- Build command `npm run build` serves as typecheck for Nuxt projects (no dedicated typecheck script)
- Comments in migrations improve maintainability and explain the purpose of each change
