## Codebase Patterns
- Migration pattern: Use `ADD COLUMN IF NOT EXISTS` for safe migrations that can run multiple times
- CHECK constraints: Use DO block to add constraints conditionally (prevents errors on re-run)
- Type patterns: Database types follow Row/Insert/Update structure with nullable fields having `| null` in Row and `?: type | null` in Insert/Update
- Build command `npm run build` serves as typecheck for Nuxt projects (no dedicated typecheck script)
- Layer system: The app has 3 layers per illusion (intellectual, emotional, identity), stored as illusion_layer in conversations table
- Identity Layer 3 is the final session before ceremony: illusion='identity' (illusion #5) AND illusionLayer='identity' (layer 3)
- The complete-session endpoint controls program_status transitions: in_progress → ceremony_ready → completed
- Token detection pattern: Watch messages array with deep:true, check last message content.includes('[TOKEN]'), emit event on detection
- Prompt builder pattern: Export function that accepts context object and returns string prompt
- Session page uses VoiceSessionView component (auto-imported from components/voice/SessionView.vue)
- Conversations API endpoint returns full conversation data including illusion_layer field
